CONFIDENCE SCORE CANDIDATES
===========================
Ranked from most useful to least useful for evaluating agent quality.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 1 — NLI Validation Score
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  How confident the system was that the user's response satisfied the current
  goal. NLI (Natural Language Inference) is a separate model that asks:
  "Given what the agent asked, did the user's answer confirm it, deny it,
  or neither?" The score reflects how certain the model was about that judgment.

Why it's most useful:
  This is the most direct measure of whether the agent correctly understood
  what the user meant in context. A low score here means the agent may have
  misinterpreted an answer and moved forward incorrectly — which is exactly
  the kind of error human annotation should catch.

Where it appears in the log:
  director_state → last_behavior → log

Examples:
  "NLI Value Validation v9 for goal "good time to schedule":
   User: let me think => User agreed to schedule => entailment (0.9829)"

  "NLI Validation v9 for goal meeting slot:
   User: No => User rejected meeting time => entailment=0.9990"

  "NLI Value Validation v8 for goal "speaker verification":
   User: Hi. This is Ryan => User name is Ryan Hafey => entailment (1.0)"

Score range: 0.0 to 1.0 (higher = more confident)
Also note: The label before the score (entailment / neutral / contradiction)
describes the relationship, and the number is the confidence in that label.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 2 — Intent Confidence  ← CURRENTLY USED
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  How confident the system was in identifying what the user meant
  (their "intent") — regardless of the current goal context.
  It classifies the user's utterance into a named intent category.

Why it's second:
  Useful and already proven, but it only measures whether the utterance was
  recognised, not whether the agent's response to it was correct.
  A high intent confidence doesn't rule out the agent responding poorly.

Where it appears in the log:
  director_state → last_behavior → log

Examples:
  "detected intent "hello" as "hello?" 0.96%"

  "detected intent "confirm" as "yes, it is" 0.96%"

  "detected hard-coded intent "confirm" as "Yes." 1.0%"

Score range: 0.0% to 1.0% (note: displayed as a percentage in the log,
stored as a decimal — e.g. "0.96%" is stored as 0.96, not 0.0096)


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 3 — Goal Trigger Confidence
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  How confident the system was that the user's message should trigger a
  specific goal in the agent's script. While intent confidence classifies
  what the user said, this measures whether that maps to the right next
  step in the conversation flow.

Why it's third:
  Adds a layer of context beyond intent — you can see both whether the
  intent was recognised AND whether the right goal was triggered. However,
  it often has the same value as intent confidence, making it somewhat
  redundant as a standalone score.

Where it appears in the log:
  director_state → last_behavior → log

Examples:
  "Goal "speaker verification" considered by TRIGGER "hello" 0.96%"

  "Goal "call screener handling" considered by TRIGGER
   "Hi, If you record your name..." 1.0%"

Score range: 0.0% to 1.0%


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 4 — Voice Intent Confidence
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  The same as intent confidence, but specifically for voice-detected intents
  (a separate detection pipeline optimised for spoken language, as opposed
  to the text-based intent classifier).

Why it's fourth:
  Provides a useful secondary signal — if text intent and voice intent
  disagree or have very different confidence scores for the same utterance,
  that itself is a meaningful flag. However, as a standalone score it's
  largely equivalent to intent confidence.

Where it appears in the log:
  director_state → last_behavior → log

Examples:
  "detected voice_intent "%continue%" as "yes" 1.0%"

  "detected voice_intent "%sorry_what%" as "sorry what?" 0.93%"

Score range: 0.0% to 1.0%


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 5 — Satisfaction Score
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  A running accumulated score across the entire call that goes up when the
  conversation progresses well and drops when the user pushes back, refuses,
  or repeats themselves. It reflects the overall health of the call at each
  turn, not the quality of any single exchange.
  Also available directly as a numeric field in director_state.satisfaction
  (no log parsing needed).

Why it's fifth:
  Useful for call-level analysis (e.g. identifying calls that were going
  well until a specific point), but less useful for per-pair annotation
  because it accumulates over time — a high score on turn 8 doesn't mean
  turn 8 specifically went well, just that the call was going well overall
  up to that point.

Where it appears:
  Log line:  director_state → last_behavior → log
  Direct field: director_state → satisfaction  (numeric, no parsing needed)

Examples:
  "Satisfaction=0.25 (+0.25) by user turn"    ← early in call, low but rising
  "Satisfaction=11.75 (-0.5) by user turn"    ← late in call, high but dipping
  "Satisfaction=4.7 (+0.2) by Agent turn=are-you-there"

Score range: Unbounded accumulator (not 0–1). Increases with progress,
decreases with resistance. Meaningful as a trend, not an absolute value.


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
RANK 6 — Prompt Validation Confidence
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

What it measures:
  How confident the system was that the agent's own question (prompt) was
  well-formed and appropriate for the current goal. This evaluates the
  agent's output, not the user's input.

Why it's least useful:
  It measures the agent's question quality rather than the user's response
  quality, making it less directly relevant to annotation tasks focused on
  whether the agent understood the user correctly. Also appears relatively
  rarely in the logs compared to the other candidates.

Where it appears in the log:
  director_state → last_behavior → log

Examples:
  "Goal "inbound contact name" prompt validated at 100.0% confidence
   with sentence: who am i speaking with"

  "Goal "speaker verification" prompt validated at 100.0% confidence
   with sentence: am i speaking to kyle glover"

Score range: 0.0% to 100.0% (displayed as a percentage)
